# 🏡 Project Hearth（我们的家）
# Page Specification

版本：v1.0

---

# 0. 页面设计总原则

《我们的家》不是一个功能集合。

它是一栋房子。

每个页面都是一个空间。

用户不是点击功能。

而是在房子里移动。

空间关系：

```
          🏡 Home
              |
  ┌───────────┼───────────┐
  ↓           ↓           ↓
🛋 客厅      📖 卧室      📚 书房
Chat        Diary       Studio
              |
            🌱 花园
              |
           🖼 相册
```

---

# 1. Home 首页（玄关）

## 定位

首页是进入家的第一空间。

不是 Dashboard。

不是功能列表。

它负责：

让用户产生：

"我回来了。"

## 页面目标

用户打开后看到：

- 当前时间
- 今日状态
- AI状态
- 最近发生的事情
- 家的变化

## 页面结构

### 第一部分：欢迎区域

内容：

欢迎回家。

今天是：

2026年7月10日

星期五

下面：

我们已经一起生活：

382天

设计：

大标题。

大量留白。

### 第二部分：今日状态

展示：

天气。

时间。

简单状态。

例如：

☀️

成都

26°

下午阳光很好。

不要做天气APP。

### 第三部分：AI状态

展示：

问现在在哪里。

例如：

问现在在：

书房

正在整理今天的回忆。

### 第四部分：家的变化

显示：

最近发生的小事情。

例如：

🌱

花园长高了一点。

📷

收藏了一张新的照片。

💌

收到一条留言。

### 第五部分：进入聊天

不是最大按钮。

而是一张邀请卡。

例如：

坐下来聊一会？

进入客厅 →

# 2. Chat 客厅

## 定位

聊天空间。

不是AI窗口。

而是：

两个人坐下来交流。

## 页面目标

保留：

- 连续感
- 情绪
- 上下文

## 顶部

显示：

客厅

问

正在这里陪你

不要：

AI Assistant。

## 消息设计

不要传统气泡。

形式：

时间线。

例如：

```
下午 16:20

惠惠：

今天有点累。

问：

是不是论文又让你头疼了？
```

## AI状态

回复前：

显示：

……

问正在想。

或者：

问翻了一下以前的记录。

## 输入框

Placeholder:

不要：

输入消息

改：

想和问聊点什么？

# 3. Diary 日记（卧室）

## 定位

私人记录空间。

安静。

慢。

## 页面目标

记录：

今天。

## 页面结构

顶部：

7月10日

今天怎么样？

心情选择：

😊 😌 😐 😔

内容区域：

像纸。

不是编辑器。

## AI辅助

不是代写。

可以：

整理。

总结。

陪伴。

例如：

用户写：

"今天很累"

AI：

"今天辛苦了，要不要早点休息。"
